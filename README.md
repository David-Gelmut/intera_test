# Интеграция с Яндекс.Картами (Тестовое задание)

## Технологический стек

- **Laravel 13**
- **PHP 8.4+**
- **MySQL 8**
- **Redis**
- **Nginx**
- **Docker**


## Запуск сервиса
1. Подготовка окружения

```bash 
  # Клонирование репозитория
  git clone https://github.com/David-Gelmut/intera_test.git
  
  # Создание конфигурационного файла
  cp .env.example .env
  
  # Установка зависимостей Backend
  composer install
  
```

2. Запуск в Docker

```bash 
  # Сборка и запуск контейнеров
  docker-compose up -d --build

  #Остановка контейнеров
  docker-compose down

  # Генерация ключа приложения
  docker-compose exec -it app php artisan key:generate
```

3. База данных

```bash 
  docker-compose exec -it app php artisan migrate
```
```bash
  # Тестовый пользователь  admin@example.com password
  docker-compose exec -it app php artisan db:seed
  
  
 ```
4. Установите зависимости фронтенда, бэкенда и запустите сборку:
   ```bash
    npm install
    npm run dev
   ```
5. Для Linux:
```bash
  sudo chmod -R 775 storage bootstrap/cache && sudo chown -R www-data:www-data storage bootstrap/cache
```


Сервис доступен локально по http://localhost:8000/

Для теста на проде https://yandex.ru/maps/org/burger_king/9794836194


## Архитектурные решения и обоснование

### 1. Метод парсинга: Headless-браузер вместо разбора внутренних API-запросов
Вместо использования разбора внутренних API-запросов(что было бы оптимальнее), был выбран Headless-браузер.
Разбор внутренних API-запросов не дал результата, так как Яндекс динамически меняет токен, передаёт дополнительные параметры, ошибка в которых выдаёт моментально Bad Request.

### 2. Кэширование на стороне бэкенда
Парсинг ~600 отзывов в режиме реального времени происходит в фоне. При последующем обращении данные подтягиваются из базы.
Есть возможность кэшировать в Redis докер контейнере, но на данный момент не реализовано, а реализована пагинация по 50 отзывов компании на странице.

### 3. Что ещё можно сделать.
